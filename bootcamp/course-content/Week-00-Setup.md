# Week 00 – Bootcamp Setup

## Day 01: Orientation and learning workflow

### Lesson Content
- Program goals, grading, and expectations (attendance, deadlines, feedback).
- How to learn programming: spaced repetition, deliberate practice, and iteration.
- Build a personal learning system (daily plan, reflection, tracking).

### Instructor Demo
- Create a learning journal in Markdown and organize by week.
- Write a SMART goal and break it into weekly targets.
- Map job requirements to skills you will learn in this bootcamp.

### Lab / Workshop
- Create a personal learning journal repo and add Week 00 notes.
- Write a daily schedule for the first two weeks.
- Draft a one-paragraph goal statement and commit it.

### Assessment / Checkpoint
- Submit your journal repo link and goal statement.

## Day 02: Environment setup and verification

### Lesson Content
- Install and verify Visual Studio, .NET SDK, and VS Code.
- Install SQL Server Express + SSMS and validate connections.
- Command line basics: pwd, ls, cd, mkdir, touch, code, dotnet.

### Instructor Demo
- Create a Hello World console app with dotnet CLI.
- Run a test SQL query in SSMS (SELECT @@VERSION).
- Open a project in VS Code and run it from the terminal.

### Lab / Workshop
- Install all required tools and capture version output.
- Create and run a Hello World console app.
- Connect to SQL Server and run a simple query.

### Assessment / Checkpoint
- Provide screenshots or logs confirming setup success.

## Day 03: GitHub account + SSH setup

### Lesson Content
- GitHub fundamentals: repositories, commits, and README files.
- SSH keys vs HTTPS and why SSH is preferred.
- Basic Git workflow: clone, add, commit, push.

### Instructor Demo
- Generate SSH keys and add them to GitHub.
- Create a repo and push a README.
- Clone the repo locally and add a second commit.

### Lab / Workshop
- Create a repo named bootcamp-journal.
- Commit a README and a journal entry.
- Push via SSH.

### Assessment / Checkpoint
- Submit repo link with at least two commits.

## Day 04: File structure and project organization

### Lesson Content
- Common project layout: src/, docs/, tests/, assets/.
- Naming conventions for files and folders.
- Markdown basics: headings, lists, code blocks, links.

### Instructor Demo
- Create a sample project folder with docs and src.
- Write a short README with project goals and setup steps.
- Commit and push structure changes.

### Lab / Workshop
- Build a template repo structure for future projects.
- Create a README template file.

### Assessment / Checkpoint
- Submit repo structure and README template.

## Day 05: Workflow rehearsal

### Lesson Content
- Daily workflow: plan → build → test → commit → reflect.
- Breaking tasks into small, trackable items.
- Preparing for Week 01 (variables, types, I/O).

### Instructor Demo
- Plan a 3-task mini project and write a checklist.
- Make a small change and commit it with a clear message.

### Lab / Workshop
- Create a Week 01 plan in your journal.
- Commit the plan to GitHub.

### Assessment / Checkpoint
- Week 01 plan and workflow checklist.
